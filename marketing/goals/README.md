# goals/

KPIs, measurement plans, sprint cycle results. **Read this folder FIRST before producing any output** — every skill should be aligned to a specific goal.

**What goes here:**
- `MMYY-scope.md` — KPI definitions, baselines, targets, deadlines for a measurement period (typically a quarter)
- `measurement.md` (system file) — how each KPI is measured (sources: GA4, Search Console, CRM, dashboards; cadence; owner)
- `cadence.md` (system file) — sprint length (1 or 2 weeks) and review cadence
- `MMYY-NN-cycle.md` — per-sprint result file (NN = sprint number within the month, results + hypotheses for next sprint)

**No skill produces this folder automatically — you author it manually from your engagement contract or quarterly planning.**

**Why this matters:** without explicit KPIs, every skill output is decorative. With them, every skill output is testable against "did this move the metric we agreed to move?"
