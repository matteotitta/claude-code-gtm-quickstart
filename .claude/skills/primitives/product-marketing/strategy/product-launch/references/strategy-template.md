## Field specifications

### Feature category definitions

| Category | Definition | Examples |
|----------|------------|----------|
| **Major feature** | New capability that changes core workflow | New product line, major platform expansion |
| **Integration** | Connection to third-party tool | Salesforce sync, Slack integration |
| **Enhancement** | Improvement to existing capability | Performance boost, UX redesign |
| **Platform** | Infrastructure or foundational change | API v2, new authentication |

### Launch tier criteria

| Tier | Criteria | Asset scope |
|------|----------|-------------|
| **Major** | New revenue stream, market expansion, platform shift | All 18 assets |
| **Significant** | Core feature, key integration, pricing change | 12-15 assets |
| **Standard** | Feature improvement, workflow enhancement | 8-10 assets |
| **Minor** | Bug fix, minor UX improvement | 4-6 assets |

### Objective setting guidelines

| Metric type | Baseline source | Target multiplier |
|-------------|-----------------|-------------------|
| Pipeline | Previous launch + seasonality | 1.2-2x depending on tier |
| Adoption | Similar feature adoption curves | Adjust for ICP size |
| Awareness | Previous campaign benchmarks | Match tier to budget |
