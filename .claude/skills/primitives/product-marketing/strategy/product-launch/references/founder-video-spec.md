# Founder video specification

Demo video script structure and production guidance for product launch announcements.

## Video overview

| Element | Specification |
|---------|---------------|
| **Length** | 2-3 minutes (ideal: 2:30) |
| **Format** | Mix of talking head + screen recording |
| **Speaker** | Founder, CEO, or Product lead |
| **Platform** | LinkedIn (native), YouTube, embedded in blog |
| **Purpose** | Build authority, demo feature, drive action |

---

## Script structure

### Hook (0:00-0:15)

**Purpose:** Stop the scroll, create curiosity

**Patterns:**

| Pattern | Example |
|---------|---------|
| Problem statement | "[Pain point]. Today, we're fixing that." |
| Bold claim | "We just shipped the feature we've been working on for 8 months." |
| Customer voice | "The #1 request from our customers was [X]. So we built it." |
| Contrarian | "Most [category] tools do [X]. We think that's wrong." |

**Script template:**
```
[Opening line — problem or announcement]
[Transition to what we built — 1 sentence]
```

### Context (0:15-0:45)

**Purpose:** Establish why this matters now

**Components:**
- Why we built this (customer need or market gap)
- What changed or what we learned
- Brief description of the feature

**Script template:**
```
"For the past [X months], we've heard from customers that [pain point].

They were [current workaround], which meant [consequence].

So we built [Feature name] — [one sentence description]."
```

### Demo (0:45-2:00)

**Purpose:** Show the feature in action

**Structure:**
1. Primary use case walkthrough (45-60 seconds)
2. Secondary capability highlight (20-30 seconds)
3. Key differentiator moment (15-20 seconds)

**Demo principles:**
- Show real product, not mockups
- Use realistic data (not "test" or "lorem ipsum")
- Narrate actions as you do them
- Pause on key moments
- Highlight "aha" moments explicitly

**Script template:**
```
"Let me show you how it works.

[Primary use case]
First, [action]. You'll see [result].

Now watch what happens when [trigger]. [Outcome].

[Secondary capability]
You can also [capability]. This means [benefit].

[Key differentiator]
The thing that makes this different is [differentiator].
[Show specific moment that illustrates this]"
```

### Benefits summary (2:00-2:30)

**Purpose:** Reinforce value, transition to CTA

**Components:**
- Recap of 3 key benefits
- Who this is for
- Social proof (optional)

**Script template:**
```
"With [Feature name], you can now:
- [Benefit 1]
- [Benefit 2]
- [Benefit 3]

This is especially powerful for [persona/use case].

[Optional: [Customer name] is already using it and seeing [result].]"
```

### CTA (2:30-3:00)

**Purpose:** Drive specific action

**Components:**
- Where to try/learn more
- What to do next
- Feedback invitation

**Script template:**
```
"[Feature name] is available now for [audience].

Go to [URL] to try it — [pricing/access context].

Let me know what you think. I read every response."
```

## Visual composition

### Shot types

| Shot | When to use | Duration |
|------|-------------|----------|
| **Talking head** | Hook, context, benefits, CTA | 30-45% of video |
| **Screen recording** | Demo walkthrough | 50-60% of video |
| **Picture-in-picture** | Narrating while showing product | Optional |
| **Full screen product** | Key feature moments | Brief (5-10 sec) |

### Screen recording guidelines

| Element | Specification |
|---------|---------------|
| Resolution | 1080p minimum (4K preferred) |
| Cursor | Visible, highlighted, smooth movement |
| Clicks | Pause slightly before and after |
| Scrolling | Slow and intentional |
| Zoom | On small UI elements |
| Annotations | Optional highlights/arrows |

### Talking head guidelines

| Element | Specification |
|---------|---------------|
| Framing | Chest up, slightly off-center |
| Background | Clean, branded, or blurred office |
| Lighting | Front-facing, natural or soft |
| Eye line | Camera lens (not screen) |
| Teleprompter | Acceptable if natural delivery |

## Production checklist

### Pre-production

- [ ] Script approved by PMM
- [ ] Demo environment prepared (clean data)
- [ ] Screen recording software tested
- [ ] Camera/lighting setup confirmed
- [ ] Talking points memorized or teleprompter ready

### Recording

- [ ] Record talking head segments first
- [ ] Record screen demo separately
- [ ] Capture multiple takes of key moments
- [ ] Check audio levels between segments
- [ ] Record additional b-roll (typing, product shots)

### Post-production

- [ ] Edit to target length (2-3 min)
- [ ] Add captions (required — 85% watch without sound)
- [ ] Add lower third (name, title)
- [ ] Add intro/outro bumper (optional)
- [ ] Export for multiple platforms

### Export specifications

| Platform | Format | Resolution | Aspect ratio |
|----------|--------|------------|--------------|
| LinkedIn | MP4 | 1080p | 1:1 (square) or 4:5 |
| YouTube | MP4 | 1080p or 4K | 16:9 |
| Blog embed | MP4 | 1080p | 16:9 |
| Twitter/X | MP4 | 720p | 1:1 or 16:9 |

---

## Caption requirements

### Auto-caption review

Platforms generate auto-captions — review for:
- Product/feature names (often misspelled)
- Technical terms
- Proper nouns
- Numbers and metrics

### Caption style

| Element | Guideline |
|---------|-----------|
| Line length | Max 2 lines, 32 chars each |
| Duration | 1-3 seconds per caption |
| Punctuation | Minimal — no periods at end |
| Case | Sentence case |
| Position | Bottom center, avoid UI overlap |

---

## Distribution

### Platform-specific optimization

| Platform | Optimization |
|----------|--------------|
| **LinkedIn** | Square format, captions burned in, hook in first 3 sec |
| **YouTube** | 16:9, custom thumbnail, chapters if >3 min |
| **Blog** | 16:9, autoplay off, poster image |
| **Twitter/X** | Square, under 2:20, hook in first 2 sec |

### Thumbnail guidelines

| Element | Guideline |
|---------|-----------|
| Format | PNG or JPG, 1280x720 px |
| Content | Speaker + text overlay + product hint |
| Text | Large, readable at small size |
| Branding | Logo or brand colors |
| Avoid | Clickbait, misleading imagery |

## Script example

```
[HOOK — 0:00-0:15]
"Every engineering team I talk to has the same problem.
They spend hours every week manually updating ticket status.
Today, we're fixing that."

[CONTEXT — 0:15-0:45]
"For the past six months, we've heard from customers that manual status updates are their biggest friction point.

Developers hate it. PMs chase updates. And the data is never accurate anyway.

So we built Git Sync — automatic two-way sync between your codebase and your issues."

[DEMO — 0:45-2:00]
"Let me show you how it works.

[Screen: Settings > Integrations]
First, connect your GitHub or GitLab repository. One click, takes 30 seconds.

[Screen: Issue with linked PR]
Now watch what happens. I'm going to merge this PR...

[Screen: Issue auto-closes]
And the issue closes automatically. No manual update. No stale tickets.

[Screen: Create issue from commit]
You can also go the other direction. Create an issue directly from a commit — it links automatically.

[Screen: Real-time sync indicator]
The thing that makes this different is it's bi-directional and real-time. 
Your boards always reflect what's actually happening in your codebase."

[BENEFITS — 2:00-2:30]
"With Git Sync, you can now:
- Eliminate manual status updates
- Keep your boards accurate in real-time
- Let developers stay in flow

This is especially powerful for engineering teams shipping fast.

Vercel is already using it — they eliminated 4 hours of weekly status meetings."

[CTA — 2:30-3:00]
"Git Sync is available now for all plans.

Go to linear.app/sync to set it up — takes 2 minutes.

Try it and let me know what you think. I read every response."
```

## Quality checklist

- [ ] Script follows hook-context-demo-benefits-CTA structure
- [ ] Total length is 2-3 minutes
- [ ] Hook captures attention in first 10 seconds
- [ ] Demo shows real product with realistic data
- [ ] Key differentiator is highlighted explicitly
- [ ] Benefits are summarized clearly
- [ ] CTA includes specific URL and action
- [ ] Captions are accurate and readable
- [ ] Exported for all target platforms
- [ ] Thumbnail is custom and compelling
