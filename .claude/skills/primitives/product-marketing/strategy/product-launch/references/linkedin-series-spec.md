# LinkedIn series specification

LinkedIn content structure for product launches. Generates pre-launch, launch day, and post-launch posts for founders and team amplification.

## Series structure

| Phase | Timing | Posts | Purpose |
|-------|--------|-------|---------|
| **Pre-launch teasers** | T-7 to T-1 | 2-3 | Build anticipation, announce webinar |
| **Launch day** | T-0 | 1 (founder) | Primary announcement with video |
| **Post-launch** | T+1 to T+5 | 2-3 | Webinar recap, social proof, deep dives |

Total: 5-7 LinkedIn posts per launch

---

## Pre-launch teasers (T-7 to T-1)

### Teaser 1: Problem agitation (T-7)

**Purpose:** Surface the pain point without revealing solution

**Structure:**
```
[Problem-focused hook — 1-2 lines]

[Agitate the pain with specific scenario — 2-3 lines]

[Hint at change coming — 1 line]

[Engagement question or tease]
```

**Example:**
```
Every product team I talk to has the same problem.

They spend hours every week manually updating ticket status. Developers hate it. PMs chase updates. Nobody has accurate data.

We've been working on something that might change this.

More soon. 👀
```

**Engagement tactics:**
- End with curiosity gap (no reveal)
- Ask question to prompt comments
- Use 👀 or 🔜 emojis sparingly

### Teaser 2: Webinar announcement (T-5)

**Purpose:** Drive webinar registrations

**Structure:**
```
[Announcement hook — 1 line]

[What we're launching — 1-2 lines, still vague]

[Webinar details — date, time, what attendees get]

[CTA with registration link]
```

**Example:**
```
Mark your calendar: [Date] at [Time] [Timezone]

We're launching something we've been building for 6 months.

Join me and [Co-presenter] for a live demo and AMA where we'll show you:
• [Benefit 1]
• [Benefit 2]
• [Benefit 3]

First 100 registrants get [incentive].

Link in comments 👇
```

### Teaser 3: Behind-the-scenes (T-3)

**Purpose:** Build founder authority, humanize the launch

**Structure:**
```
[Personal reflection hook — 1-2 lines]

[Story of building the feature — 2-3 lines]

[What it means for customers — 1-2 lines]

[Countdown to launch]
```

**Example:**
```
3 days out from our biggest launch this year.

Here's what I've learned building this:

[Insight 1 from development]
[Insight 2 from customer conversations]
[Insight 3 about the market]

Can't wait to show you what we've built.

Tuesday.
```

## Launch day (T-0)

### Primary announcement (Founder)

**Purpose:** Drive awareness, traffic to landing page, demo requests

**Structure:**
```
[Bold announcement hook — 1 line, make it news]

[What we launched — 2-3 sentences explaining the feature]

[Why it matters now — 1-2 sentences on timing/market]

[Key benefits — 3 bullet points max]

[Social proof if available — quote or early access result]

[CTA — specific action]

[Video embed or link to blog]
```

**Example:**
```
Today we're launching [Feature Name].

[1-2 sentences on what it does]

[1 sentence on why we built it]

Here's what this means for you:

→ [Benefit 1]
→ [Benefit 2]
→ [Benefit 3]

"[Early access quote from customer]" — [Name, Title, Company]

[CTA: Try it now / Read the full announcement / Watch the demo]

[Link]
```

**Media attachment:**
- Founder video (2-3 min demo) — highest engagement
- Product GIF — second best
- Static image — acceptable

**Engagement tactics:**
- Pin comment with link
- Reply to every comment in first 2 hours
- Tag relevant people (sparingly)

## Team sharing templates

Templates for team members to amplify launch posts. Customize voice but maintain message consistency.

### Template 1: Feature-focused

```
We just shipped [Feature Name] at [Company].

I've been working on this for [X months] and here's why it matters:

[Personal perspective on the problem it solves]

[Link to founder post or landing page]
```

### Template 2: Customer-focused

```
Excited to announce [Feature Name] is live.

We built this because [Customer segment] kept telling us:
"[Pain point in their words]"

Now they can [Key capability].

Check it out: [Link]
```

### Template 3: Behind-the-scenes

```
Launch day at [Company] 🚀

Here's what it took to ship [Feature Name]:
• [X] months of development
• [X] customer interviews
• [X] iterations

Proud of this team.

Full announcement: [Link]
```

### Template 4: Sales/CS angle

```
Starting today, I can finally help customers with [Problem].

[Feature Name] just launched and it [Key benefit].

If you're dealing with [Pain point], let's talk.

Details: [Link]
```

## Post-launch series (T+1 to T+5)

### Post 1: Webinar recap (T+1)

**Purpose:** Capture those who missed, extend reach

**Structure:**
```
[Recap hook — 1 line]

[Key takeaways from webinar — 3-4 bullets]

[Most asked question + answer]

[CTA to recording or landing page]
```

### Post 2: Social proof / early results (T+3)

**Purpose:** Build credibility with proof

**Structure:**
```
[Results hook — specific metric if available]

[What customers are saying — quote or paraphrase]

[Unexpected use case or outcome]

[Invitation to try]
```

### Post 3: Deep dive / educational (T+5)

**Purpose:** SEO, thought leadership, continued engagement

**Structure:**
```
[Educational hook related to feature category]

[Framework or insight from building feature]

[How feature applies to the insight]

[Soft CTA]
```

## Copy guidelines

### Voice application

Pull from TOV guidelines:

| Element | Application |
|---------|-------------|
| Person | Match founder's established voice (I/we) |
| Formality | Conversational, direct |
| Vocabulary | Customer language from ICP research |
| Hooks | Match established hook patterns |

### Formatting rules

| Rule | Specification |
|------|---------------|
| Length | 150-300 words (sweet spot for engagement) |
| Line breaks | After every 1-2 sentences |
| Bullets | Arrow (→) for benefits, dot (•) for lists |
| Emojis | Max 2-3 per post, use sparingly |
| Hashtags | 0-2 max, in body not end |
| Links | In first comment, not body |

### Engagement optimization

| Tactic | How |
|--------|-----|
| Hook in first line | Must stop scroll |
| Mobile-first | First 3 lines visible in feed |
| Comment prompt | End with question or invitation |
| Pin comment | Add link immediately |
| Response window | Reply to all comments in first 2 hours |

## Media specifications

### Video

| Spec | Requirement |
|------|-------------|
| Length | 2-3 minutes max |
| Format | Square (1:1) or Vertical (4:5) |
| Resolution | 1080p minimum |
| Captions | Required (85% watch without sound) |
| Thumbnail | Custom, not auto-generated |

### Image/GIF

| Spec | Requirement |
|------|-------------|
| Dimensions | 1200x1200 (square) or 1200x628 (landscape) |
| Format | PNG for graphics, GIF for animations |
| File size | Under 5MB |
| Text overlay | Minimal, readable at mobile size |
