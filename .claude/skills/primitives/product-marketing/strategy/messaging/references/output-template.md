## Linkage requirements

The messaging library must maintain clear linkages between components:

```
Status Quo → Pain Points → Capabilities → Features
                 ↓              ↓
         Consequence      Benefits ← Customer Quotes
           Chains              ↓
                        Emotional/Social
```

### Validation checklist

Before finalizing output:

| Linkage | Requirement | Check |
|---------|-------------|-------|
| Pain point → Consequence chain | Every pain point has 1st → 2nd → 3rd order impacts | [ ] |
| Pain point → Capability | Every pain point references which capability addresses it | [ ] |
| Capability → Feature | Every capability references its branded feature name | [ ] |
| Feature → Capability | Every feature references which capability it powers | [ ] |
| Benefit → Capability/Feature | Every benefit links to related capability or feature | [ ] |
| Quote → Benefit | Every customer quote is embedded within a benefit | [ ] |
| Cost of inaction → Pain point | Every urgency statement links to a pain point | [ ] |

## Quote handling rules

### Verbatim requirement

Customer quotes must be:
- Copied exactly as they appear in the source
- Not paraphrased, summarized, or "improved"
- Not combined from multiple sources
- Attributed with all available information

### Attribution format

```markdown
> "[Exact quote text]"

- **Customer**: [First Last], [Title]
- **Company**: [Company Name]
- **Context**: [e.g., "On switching from Jira", "G2 review"]
- **Quote source**: [URL] | **Accessed**: [YYYY-MM-DD]
```

### Missing attribution

If parts of attribution are missing:
```markdown
- **Customer**: [First Last], Title not provided
- **Company**: Company not disclosed
```

### No quote available

When no relevant quote exists for a benefit:
```markdown
**Customer proof**: Not available — no testimonial found for this specific benefit. Recommend customer interviews.
```
