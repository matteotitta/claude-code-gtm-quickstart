# Format Guidelines by Platform

## LinkedIn Article (Long-form)

| Element | Guideline |
|---------|-----------|
| **Word count** | 800-1,500 words |
| **Title** | 50-100 characters, specific and benefit-driven |
| **Opening** | Hook within first 2 sentences, thesis by paragraph 2 |
| **Structure** | 3-5 sections with clear headlines |
| **Paragraphs** | 2-4 sentences each |
| **Images** | Cover image required, 1-2 supporting images optional |
| **CTA** | Soft ask at end (comment, follow, connect) |

### LinkedIn Article Formatting

```
# Headline (Sentence case)

[Hook paragraph — 2-3 sentences that create tension or curiosity]

[Context paragraph — why this matters now]

[Thesis statement — your governing thought, clearly stated]

---

## Section 1 Headline

[2-4 paragraphs expanding key line 1]

## Section 2 Headline

[2-4 paragraphs expanding key line 2]

## Section 3 Headline

[2-4 paragraphs expanding key line 3]

---

## The Takeaway

[Synthesis paragraph — what this all adds up to]

[Implication paragraph — what the reader should do differently]

[Soft CTA — invite discussion, ask a question]
```

## Blog Post

| Element | Guideline |
|---------|-----------|
| **Word count** | 1,500-2,500 words |
| **Title** | SEO-conscious, specific, under 60 characters for search |
| **Meta description** | 150-160 characters |
| **Opening** | Hook + context + thesis within first 200 words |
| **Structure** | 4-7 sections with H2 headlines |
| **Subheads** | H3 for subsections where needed |
| **Lists** | Use for scanability, 3-7 items |
| **Images** | Hero image + 1-2 supporting visuals or frameworks |
| **CTA** | Can be stronger (newsletter signup, consultation, etc.) |

### Blog Post Formatting

```
# Title (SEO-optimized, sentence case)

*[Optional: Reading time estimate or topic tag]*

[Hook paragraph — pattern interrupt or surprising statement]

[Context paragraph — the problem or situation]

[Thesis paragraph — your governing thought and roadmap]

---

## Section 1: [Key Line 1 as Headline]

[Lead paragraph stating the argument]

[Evidence paragraph with examples/data]

[Implication paragraph — so what?]

### Subsection (if needed)

[Additional detail]

---

## Section 2: [Key Line 2 as Headline]

[Same structure as above]

---

## Section 3: [Key Line 3 as Headline]

[Same structure as above]

---

## What This Means for You

[Synthesis — connecting the key lines]

[Specific actions the reader can take]

[CTA — clear next step]

---

*[Optional: Author bio, related posts, newsletter signup]*
```

## Guest Post / Byline

| Element | Guideline |
|---------|-----------|
| **Word count** | 2,000-3,500 words (publication dependent) |
| **Title** | Pitch multiple options to editor |
| **Opening** | Strong hook, establishes authority early |
| **Structure** | 5-8 sections, deeper than blog |
| **Tone** | Slightly more formal, wider audience |
| **Credentials** | Weave proof points naturally into content |
| **CTA** | Follows publication guidelines (often soft or none) |
| **Bio** | Usually 2-3 sentences with one link |

### Guest Post Considerations

- Research publication's style before writing
- Match their headline and subhead conventions
- Adapt TOV to be slightly more accessible (broader audience)
- Include credentials naturally ("In working with 45+ Series A startups, I've seen...")
- Provide editor with suggested social copy and images

## Newsletter Essay

| Element | Guideline |
|---------|-----------|
| **Word count** | 800-1,200 words |
| **Opening** | Personal hook, often anecdote-led |
| **Structure** | Less formal, more conversational flow |
| **Tone** | Most personal of all formats |
| **Sections** | 2-4 loose sections, can be more flowing |
| **CTA** | Reply to this email, share, etc. |

### Newsletter Formatting

```
[Personal opening — story, observation, or recent experience]

[Bridge to insight — how this connects to a broader point]

[The main idea — governing thought, stated conversationally]

[The "here's why" — key supporting arguments, woven together]

[The "so what" — implication for reader]

[Sign-off — personal, invites response]

— Matteo
```

## Executive Summary / One-Pager

| Element | Guideline |
|---------|-----------|
| **Word count** | 400-600 words |
| **Purpose** | Distill longer piece for quick consumption |
| **Structure** | Governing thought + 3-4 bullets + implication |
| **Tone** | Direct, no fluff |

### One-Pager Format

```
## [Topic] — Executive Summary

**The core insight:** [Governing thought in 1-2 sentences]

**Key points:**

→ [Key line 1 compressed to 1-2 sentences]

→ [Key line 2 compressed to 1-2 sentences]

→ [Key line 3 compressed to 1-2 sentences]

**The implication:** [What this means for the reader]

**Want the full version?** [Link to complete piece]
```

## Format Selection Guide

| If the goal is... | Use this format |
|-------------------|-----------------|
| Build authority on LinkedIn | LinkedIn Article |
| SEO and evergreen content | Blog Post |
| Reach new audience | Guest Post |
| Nurture existing subscribers | Newsletter Essay |
| Share key insights quickly | Executive Summary |

---

## Universal Formatting Rules

Regardless of platform:

1. **Front-load value** — First 100 words must hook and deliver
2. **Use white space** — Dense walls of text lose readers
3. **Scannable structure** — Headlines, bullets, bold text for skimmers
4. **One idea per paragraph** — Break at thought boundaries
5. **End with action** — Tell reader what to do next
