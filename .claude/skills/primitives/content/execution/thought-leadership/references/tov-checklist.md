# TOV Quick-Reference Checklist

Use this checklist in Phase 3 to ensure thought leadership content matches your team voice.

---

## Voice Attributes Checklist

### ☐ Operator Authority
- [ ] Uses first-person "I" (not "we" unless referring to team)
- [ ] References specific tools (Clay, Octave, n8n, Figma, etc.)
- [ ] Mentions specific client contexts (anonymized if needed)
- [ ] Includes hands-on language ("when I ran...", "I've seen this...")
- [ ] Draws from real experience, not theoretical knowledge

### ☐ Prescriptive Confidence
- [ ] Makes direct recommendations without hedging
- [ ] Uses imperatives ("Do this", "Stop doing that")
- [ ] Challenges conventional wisdom explicitly
- [ ] Takes a stance (even if uncomfortable)
- [ ] Avoids: "might want to consider", "perhaps", "it depends" (unless genuinely nuanced)

### ☐ Framework-Driven Clarity
- [ ] Complex ideas organized into visual frameworks where helpful
- [ ] Uses matrices, layers, quadrants, or numbered steps
- [ ] Frameworks are original or attributed
- [ ] Structure aids understanding (not decoration)

### ☐ Candid Partnership
- [ ] Acknowledges objections and limitations honestly
- [ ] Addresses counterarguments directly
- [ ] Balances confidence with appropriate self-awareness
- [ ] No defensive tone or dismissiveness

### ☐ Specificity Over Superlatives
- [ ] Uses numbers, percentages, timeframes
- [ ] Names tools, frameworks, methodologies
- [ ] References real companies or roles (anonymized if needed)
- [ ] Avoids vague adjectives: amazing, incredible, revolutionary
- [ ] Proof points are quantified where possible

### ☐ Conversational Warmth
- [ ] Uses casual language where appropriate
- [ ] Includes occasional personal asides
- [ ] Feels like talking to a smart peer
- [ ] Not corporate or stiff
- [ ] Contractions used naturally (I'm, you're, it's)

---

## Formatting Checklist

### ☐ Typography
- [ ] Em dashes with spaces: " — " (not "--" or "—")
- [ ] Numbers as numerals: 5 hours, $100M, 3 steps
- [ ] "More than" uses +: 1600+, 45+
- [ ] Sentence case for all headlines (not Title Case)

### ☐ Lists
- [ ] Arrow (→) for sequences, processes, flows
- [ ] Checkmark (☑) for completed items, results, wins
- [ ] Bullet (•) for unordered lists
- [ ] Numbered (1. 2. 3.) for ordered steps

### ☐ Structure
- [ ] Short paragraphs (2-4 sentences)
- [ ] Mix of sentence lengths (short punchy + occasional longer)
- [ ] Clear section breaks
- [ ] White space for readability

## Banned Words Checklist

Scan final draft and remove/replace:

| Banned | Replace With |
|--------|--------------|
| solutions | [specific offering] |
| leverage | use |
| synergy | [describe the actual benefit] |
| scalable | [describe how it scales] |
| best-in-class | [specific proof of quality] |
| game-changer | [describe the actual change] |
| innovative | [describe what's actually new] |
| cutting-edge | [describe the specific capability] |
| very | [delete or use precise language] |
| really | [delete or use precise language] |
| basically | [delete] |
| just | [delete unless grammatically needed] |

## 100 Posts Test

Ask: "If I published 100 posts like this, would the aggregate feel authentic?"

### Passes if:
- [ ] Could only have been written by someone with Matteo's specific experience
- [ ] Makes claims that can be backed by real work
- [ ] Takes positions that might not appeal to everyone
- [ ] Has specific details that couldn't be guessed

### Fails if:
- [ ] Could have been written by any B2B marketer
- [ ] Is generic wisdom with no distinctive POV
- [ ] Avoids any controversial positions
- [ ] Lacks specificity or proof points

---

## Quick Fixes

| Issue | Fix |
|-------|-----|
| Too corporate | Add "I" statements, specific examples |
| Too generic | Add numbers, tool names, client context |
| Too hedging | Delete qualifiers, make declarative |
| Too long sentences | Break at commas, use periods |
| Too many adjectives | Replace with specifics |
| Missing warmth | Add personal aside or acknowledgment |

---

## Final Read-Aloud Test

- [ ] Read the entire piece aloud
- [ ] Mark any sentence that feels awkward to say
- [ ] Note anywhere you'd naturally pause differently
- [ ] Adjust until it flows like natural speech
