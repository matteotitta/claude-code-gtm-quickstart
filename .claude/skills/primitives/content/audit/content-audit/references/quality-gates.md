## Anti-Hallucination Guardrails

1. **Only report what you can see.** Don't invent metrics, estimate engagement, or guess at strategy.
2. **Mark data access limits.** If you can't access full metrics, note "Limited data — requires API access".
3. **Source every finding.** Link to specific posts, pages, or content pieces.
4. **Acknowledge uncertainty.** Use "appears to," "approximately," "based on visible data".
5. **Don't assume intent.** Report patterns, don't claim to know why they exist.
6. **Count manually if needed.** Better to say "~15 posts" than guess a percentage.

## Quality Checklist (Pre-Delivery)

### Data Quality
- [ ] All channels discovered and documented
- [ ] Access limitations noted
- [ ] Specific numbers (not vague descriptions)
- [ ] Sources linked for key findings
- [ ] Time range consistent across analysis

### Analysis Quality
- [ ] Cadence calculated per channel
- [ ] Pillar mix assessed with percentages
- [ ] Gaps identified with specific recommendations
- [ ] Success patterns documented with examples
- [ ] Scorecard completed

### Strategy Handoff Quality
- [ ] Content-strategy input summary completed
- [ ] Current baseline documented
- [ ] Recommended focus areas clear
- [ ] Voice patterns summarized for TOV skill

### Format Quality
- [ ] Output header with date and font specification
- [ ] Tables properly formatted
- [ ] Executive summary provides actionable overview
- [ ] Recommendations prioritized (quick wins, strategic, long-term)

## Anti-Examples (What to Avoid)

| Bad Pattern | Why It's Bad | Better Approach |
|-------------|--------------|-----------------|
| "They post regularly" | No specifics | "3.2 posts/month, every Tuesday" |
| "Good engagement" | Unmeasurable | "Avg 45 likes, 12 comments per post" |
| "Could improve content" | No direction | "Add 2 case studies/month to fill promotional gap" |
| "Nice voice" | Not actionable | "First-person, conversational, uses rhetorical questions" |
